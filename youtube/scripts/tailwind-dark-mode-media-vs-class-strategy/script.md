# Tailwind Dark Mode: media vs class

**Tone:** Serious / Educational
**Duration:** ~30 seconds

| Sentence | Type |
|----------|------|
| Pick the wrong Tailwind dark mode strategy and you lose all user control. | Hook |
| Here is which one to use and why the difference matters. | Title Reveal |
| Tailwind has two dark mode strategies: `media` and `class`. | Introduction |
| `media` reads the OS preference automatically — no JavaScript, no manual toggle. | Body – Point 1 |
| `class` mode activates when a `dark` class exists on the html element. | Body – Point 2 |
| One line of JavaScript gives you a fully controllable user-facing toggle. | Body – Point 3 |
| No toggle needed? Use `media`. Need a toggle? Use `class`. | Conclusion |
| Open your tailwind.config.js — are you using the right strategy for your UX? | CTA |
