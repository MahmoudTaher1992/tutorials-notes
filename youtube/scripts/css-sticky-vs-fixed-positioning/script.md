# CSS sticky vs fixed: Know the Difference

**Tone:** Serious / Educational
**Duration:** ~30 seconds

| Sentence | Type |
|----------|------|
| Your sticky header stops working and you have no idea why. | Hook |
| Here is the CSS rule that silently breaks position: sticky every time. | Title Reveal |
| Both sticky and fixed keep an element visible while the user scrolls. | Introduction |
| Fixed is viewport-relative — it ignores the document flow entirely. | Body – Point 1 |
| Sticky stays within its parent container and scrolls with the document. | Body – Point 2 |
| Any ancestor with overflow hidden or scroll will silently break sticky. | Body – Point 3 |
| If sticky stops working, open DevTools and check parent overflow values. | Conclusion |
