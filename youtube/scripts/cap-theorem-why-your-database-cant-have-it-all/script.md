# CAP Theorem: Pick Two

**Tone:** Serious / Educational
**Duration:** ~30 seconds

| Sentence | Type |
|----------|------|
| Every distributed database decision is secretly a trade-off you cannot avoid. | Hook |
| Here is what the CAP theorem actually means for your architecture. | Title Reveal |
| CAP stands for Consistency, Availability, and Partition Tolerance. | Introduction |
| In any real distributed system, network partitions always happen eventually. | Body – Point 1 |
| So Partition Tolerance is not optional — you always have it. | Body – Point 2 |
| That means you are really choosing between Consistency and Availability. | Body – Point 3 |
| Banks choose consistency. Social media apps choose availability. | Conclusion |
| Next time you pick a database, ask: stale data or downtime — which hurts more? | CTA |
